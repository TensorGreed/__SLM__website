Capabilities

Control-rich defaults for teams shipping real SLM systems.

BrewSLM gives immediate productivity through autopilot presets while preserving full control over contracts, training behavior, benchmark policy, and deployment packaging.

Capability Clusters

What you can do end-to-end

Data Intake + Mapping

  • Ingest local, URL, HF, Kaggle sources in one interface.
  • Generate canonical schema mappings and reusable profiles.
  • Apply validation checks before training visibility.

Contract + Runtime Preflight

  • Validate task and model compatibility constraints.
  • Check runtime dependencies and memory fit before launch.
  • Choose strict/fail-open policy per environment.

Benchmark Intelligence

  • Run sampled candidate comparisons on project data.
  • Rank by quality, latency, and budget dimensions.
  • Persist benchmark traces for reproducible selection.

Training Orchestration

  • Launch local or cloud burst from shared run definitions.
  • Track checkpoints, logs, retries, and cancellation state.
  • Tune autopilot plans with manual override fields.

Artifact Packaging

  • Bundle weights, config, metrics, and run metadata.
  • Export to serving targets such as vLLM-compatible flows.
  • Version and verify bundles using checksums.

Operational Governance

  • Enforce benchmark floors as release gate conditions.
  • Track run provenance across data and model versions.
  • Standardize handoff between AI and platform teams.

Automation Playbooks

Starter patterns teams automate in the first month

Nightly Candidate Benchmark

Run sampled benchmark jobs nightly to detect model drift and ranking shifts.

$ brewslm benchmark --preset practical --schedule nightly

Weekly Retrain Gate

Train only if fresh data passes preflight and benchmark thresholds.

$ brewslm train --autopilot --require benchmark_floor=0.82

Release Candidate Export

Promote artifacts only when eval metrics beat the current production baseline.

$ brewslm export --target vllm --gate compare=production

Capability Test Drive

Validate the core stack with one short run

$ brewslm quickstart --task classify --data ./sample.csv

$ brewslm preflight --strict

$ brewslm benchmark --preset practical

$ brewslm train --autopilot

$ brewslm export --target vllm
